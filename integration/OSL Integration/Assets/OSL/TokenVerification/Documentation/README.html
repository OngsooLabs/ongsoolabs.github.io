OSL Social Sign-In Verification Server - Setup Guide

Welcome to the OSL Token Verification System. The included backend server is built with a lightweight ASP.NET Core Minimal API to provide secure token verification for Google, Facebook, Apple, and Steam sign-in flows.

This guide explains how to configure the server source, run it locally, and connect it to the Unity sample scene.

Prerequisites

Before running the server, install the .NET 8 SDK on your machine.

  • Download: https://dotnet.microsoft.com/download

Step 1: Configure Your App Secrets

For security reasons, your app secrets must never be hardcoded in the Unity client. Store them only in the server configuration.

  1. Extract Assets/OSL/TokenVerification/Server_Dotnet_Source.zip to a local working folder.
  2. Open appsettings.json in a text editor.
  3. Replace the placeholder values with the credentials from your provider dashboards.

Example configuration:

"Authentication": {
  "GoogleClientId": "YOUR_GOOGLE_CLIENT_ID",
  "FacebookAppId": "YOUR_FACEBOOK_APP_ID",
  "FacebookAppSecret": "YOUR_FACEBOOK_APP_SECRET",
  "AppleAppId": "YOUR_APPLE_BUNDLE_ID",
  "SteamAppId": "480",
  "SteamWebApiKey": "YOUR_STEAM_WEB_API_KEY_HERE"
}

Save the file after editing.

Step 2: Run the Local Server

The source archive currently includes a Windows startup script.

Windows

  • Double-click start.bat from the extracted server folder.

Alternative command line startup

If you prefer the CLI, open a terminal in the extracted server folder and run:

dotnet run

Verify that the server is running by checking for output similar to:

Now listening on: http://localhost:5000

You can then open:

  • http://localhost:5000/swagger

This allows you to test verification endpoints directly in the browser before connecting Unity.

Step 3: Connect the Unity Client

The package includes a sample scene at Assets/OSL/TokenVerification/Samples/Scene/DemoScene.unity.

The sample scene uses a runtime OnGUI interface by default so it can remain lightweight and avoid extra UI package dependencies during validation.

In the DemoSceneController component:

  • Verify Manager should reference the TokenVerifyManager component on DemoRoot
  • Token Input Field can be left empty
  • Console Log Text can be left empty

When both UI reference fields are empty, the scene automatically shows the built-in OnGUI sample UI at runtime.

To test the Unity sample:

  1. Open Assets/OSL/TokenVerification/Samples/Scene/DemoScene.unity.
  2. Set TokenVerifyManager > Server Base Url to your local or deployed server URL.
  3. Press Play.
  4. Paste a valid provider token into the runtime text box.
  5. Click one of the verify buttons: Google, Facebook, Apple, or Steam.
  6. Review the response in the on-screen log area and the Unity Console.

Implementation Example

public TokenVerifyManager verifyManager;

public void OnLoginSuccess(string myToken)
{
	verifyManager.VerifyToken(AuthProvider.Google, myToken, (response) =>
	{
		if (response.success)
		{
			Debug.Log("Server Verified! User UID: " + response.uid);
			// Proceed to the next step in your login flow.
		}
		else
		{
			Debug.LogError("Verification Failed: " + response.message);
		}
	});
}

Deploying to Production

The sample is configured for local testing with http://localhost:5000.

Before shipping your game, deploy the extracted server source to a real host such as Azure, AWS, Google Cloud, or another VPS environment. After deployment, update serverBaseUrl in TokenVerifyManager to point to your live API domain, for example:

  • https://api.yourgame.com

Troubleshooting

Port binding error

If you see an error similar to Failed to bind to address http://127.0.0.1:5000, another process is already using that port. Change the configured port in your server startup settings and use the same port in Unity.

Facebook app access token error

If Facebook verification returns an app access token error, verify that FacebookAppId and FacebookAppSecret are present and correct in appsettings.json.

Optional custom UI

If you want to use your own UI, you may assign compatible text-based components to the following fields on DemoSceneController:

  • Token Input Field
  • Console Log Text

The controller reads and writes a public text property through reflection, so it can work with supported legacy UGUI or TMP-style components without taking a hard assembly dependency in the sample script.
