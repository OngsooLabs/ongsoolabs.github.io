OngsooLabs.Api 프로젝트 지침 (PROJECT_GUIDELINE.md)

본 문서는 OngsooLabs.Api 솔루션의 아키텍처 원칙, 의존성 규칙 및 프로덕션 환경 출시를 위해 반드시 준수해야 하는 보안 통제 지침을 정의합니다. 모든 개발자는 본 지침을 엄격히 준수하여 코드를 작성해야 합니다.

1. 아키텍처 및 레이어 의존성 규칙

본 솔루션은 src/Services/<ServiceName>/ 경로 하위에 개별 백엔드 서비스를 격리하는 멀티 서비스 모노레포 구조를 따릅니다. 의존성은 언제나 고수준(Host)에서 저수준(Domain) 방향으로만 흐르는 단방향 아키텍처를 유지하며, 타 서비스와의 교차 참조는 금지됩니다.

  • WebApi ➡️ Infrastructure ➡️ Domain

📂 OngsooLabs..Domain (Core Layer)

  • 의존성 규칙: 의존성 제로 (Zero-dependency)
  • 외부 Nuget 패키지, DB 프레임워크, 웹 호스트 라이브러리를 일절 참조하지 않는 순수 C# 코드로만 작성합니다.

📂 OngsooLabs..Infrastructure (Technical Implementation Layer)

  • 의존성 규칙: 동일 서비스의 Domain 프로젝트만 참조하며, 기술 구현을 위한 외부 라이브러리(Playwright 등)를 활용합니다.

📂 OngsooLabs..WebApi (Host & Presentation Layer)

  • 의존성 규칙: 동일 서비스의 Domain 및 Infrastructure를 참조하여 DI를 조립하며, 서비스를 단독 실행 가능한 호스트로 빌드합니다.

2. 공통 코드 배치 원칙

  • 순수 C# 기반 공통 코드 ➡️ 개별 서비스의 Domain/Common/ 배치
  • 외부 라이브러리 종속 공통 코드 ➡️ 개별 서비스의 Infrastructure/Common/ 배치
  • 웹 프레임워크 종속 공통 코드 ➡️ 개별 서비스의 WebApi/Extensions/ 또는 WebApi/Helpers/ 배치

3. 프로덕션 API 보안 통제 지침 (Security Regulations)

  1. 외부 프록시 인증 필수화: WebApi 레이어는 진입 미들웨어에서 X-RapidAPI-Proxy-Secret 고유 헤더 값을 상시 검증하며, 누락되거나 불일치할 경우 비즈니스 로직(Domain)을 타지 않고 즉시 401 Unauthorized를 반환한다.
  2. 크롤러 실행 권한 제한: Dockerfile을 통한 최종 배포 이미지 빌드 시, Playwright 및 웹 드라이버는 Linux 비특권 유저(Application User) 권한으로 구동하여 호스트 커널 장악 위험을 원천 차단한다.
  3. SSRF 인풋 검증: Infrastructure 레이어의 AI Reader 엔진은 요청된 URL이 클라우드 인프라 내부 메타데이터 주소(169.254.x.x)나 로컬 호스트(localhost, 127.0.0.1)를 가리키고 있는지 검증하고, 적발 시 즉시 예외를 던진다.

4. 인프라 및 네트워크 가드레일 (AWS)

  1. 보안 그룹(Security Group) 최소 권한 허용: API 컨테이너 인바운드 규칙에서 외부 노출을 최소화하고, RapidAPI 프록시 IP 대역 화이트리스트 적용을 검토한다.
  2. SSH 포트(22) 보호: 인바운드 규칙에서 SSH 포트는 작업 인원의 공인 IP 단일 대역만 허용하도록 고정한다.

5. 공통 기술 서비스 격리 및 내부망 통신 통제 지침

  1. 공통 기술 도커 분리: 비즈니스 로직과 무관하고 범용적으로 사용 가능한 텍스트 처리, 토큰 연산 등의 기술 분석 기능은 별도의 독립된 초경량 도커 서비스로 격리하여 배포한다.
  2. 외부 노출 원천 차단: 분리된 공통 기술 서비스는 외부 포트(80, 443)를 가질 수 없으며, 오직 docker-compose로 생성된 비공개 가상 브릿지 네트워크(ongsoolabs-net) 내에서만 수신 대기한다.
  3. 내부 DNS 통신 규격: 비즈니스 도메인 서비스(예: AI Reader)가 이 공통 기술 서비스를 호출할 때는 환경 변수로 주입된 도커 내부 DNS 주소(예: http://ongsoolabs-textprocessor:8080)를 사용한다.
