Namespace OSL.SocialLogin

Classes

SocialLoginManager

Central entry point that routes login requests to the correct provider implementation. Register concrete providers at startup (typically inside a bootstrap MonoBehaviour) and call LoginAsync(ProviderKind) with the ProviderKind you want to use.

Structs

SocialLoginResult

Result payload returned by a login attempt. The Token is the value your backend must verify (ID token, access token or session ticket depending on the provider).

Interfaces

ISocialLoginProvider

Common contract every concrete provider (G / FB / A / St) implements. Async-first to play nicely with modern Unity (.NET Standard 2.1).

Enums

ProviderKind

Identifier for each supported social login backend. Names use safe abbreviations to avoid trademark conflicts.
