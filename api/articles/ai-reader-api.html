AI Reader API

AI Reader API는 웹페이지를 LLM(거대 언어 모델)이 바로 사용할 수 있는 최적의 형태로 변환하는 AI 전처리 서비스입니다.

👉 RapidAPI에서 사용해보기 (Available on RapidAPI)

핵심 가치 및 동작 구조

graph LR
A[Web Page] --> B[Content Extraction]
B --> C[Boilerplate Removal]
C --> D[Markdown Conversion]
D --> E[Metadata Analysis]
E --> F[Token Estimation]
F --> G[AI Ready Output]

주요 제공 기능

  • Markdown 변환: 웹페이지의 스타일과 불필요 요소를 걷어내고 순수 마크다운으로 변환합니다.
  • 본문 및 제목 추출: 노이즈를 필터링하고 실제 기사 또는 상품의 핵심 본문 및 제목을 추출합니다.
  • 언어 및 지표 분석: 대상 페이지의 언어 분석, 단어 수 계산, 토큰 수 계산을 지원합니다.
  • Embedding 비용 추정: 추출된 텍스트 크기에 기초한 임베딩 비용을 추정합니다.
  • 독서 시간(Reading Time) 계산: 평균 독서 속도를 기반으로 예상 시간을 제공합니다.

시스템 아키텍처 (v1.0)

  1. Browser & Context Pool: 단일 크롬 브라우저를 상시 구동하여 격리된 컨텍스트와 페이지를 풀링함으로써 초고속 렌더링을 지원합니다.
  2. Site Rule Engine: 도메인별(예: 쿠팡, 아마존, MDN 등)로 수집 전략을 분리하여 유연하게 규칙을 적용합니다.
  3. Proxy Router: 비용 효율과 수집 성공률 극대화를 위해 No Proxy ➡️ Datacenter Proxy ➡️ Residential Proxy 순으로 에스컬레이션하는 로터 필터를 지원합니다.
  4. Reader Mode & Output Options: 콘텐츠 형태에 따라 모드(article, product 등)를 지원하며, 출력 결과에 이미지나 링크 등을 포함할지 세밀하게 제어할 수 있습니다.

API 엔드포인트

AI Reader API

메서드 경로 설명
GET /v1/reader URL 파라미터를 사용한 텍스트 변환 및 분석
POST /v1/reader JSON 바디를 통한 고급 옵션(쿠키, 헤더, 프록시) 제공
POST /v1/reader/chunk 독립된 TextProcessor 마이크로서비스를 경유한 텍스트 청킹

POST /v1/reader 요청 예시

POST /v1/reader
Content-Type: application/json
X-RapidAPI-Key: <your-rapidapi-key>

{
  "url": "https://example.com",
  "mode": "article",
  "tokenizer": "openai",
  "output": {
    "content": true,
    "metadata": true,
    "tables": true,
    "codeBlocks": true,
    "images": false,
    "links": false
  }
}
Note

본 서비스는 데이터를 가공하는 파이프라인 역할을 수행하며, 생성된 결과물의 상업적 활용에 대한 모든 법적 책임은 이용자에게 있습니다. 대상 웹사이트의 이용약관 및 저작권 규정을 준수해 주세요.
