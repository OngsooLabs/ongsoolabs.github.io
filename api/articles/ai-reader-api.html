AI Reader API

AI Reader API is an AI preprocessing service that transforms web pages into an optimal format ready for immediate use by Large Language Models (LLMs).

👉 Available on RapidAPI

Core Value & Pipeline

graph LR
A[Web Page] --> B[Content Extraction]
B --> C[Boilerplate Removal]
C --> D[Markdown Conversion]
D --> E[Metadata Analysis]
E --> F[Token Estimation]
F --> G[AI Ready Output]

Key Features

  • Markdown Conversion: Strips away styling and unnecessary elements from web pages, converting them into pure markdown.
  • Content & Title Extraction: Filters out noise to extract the core text and title of actual articles or products.
  • Language & Metric Analysis: Supports language detection, word count, and token count calculations for the target page.
  • Embedding Cost Estimation: Estimates embedding costs based on the size of the extracted text.
  • Reading Time Calculation: Provides an estimated reading time based on average reading speeds.

System Architecture (v1.0)

  1. Browser & Context Pool: A single Chrome instance runs continuously from server startup. It pools isolated contexts and pages to support ultra-fast rendering.
  2. Site Rule Engine: Extraction strategies are separated by domain (e.g., Amazon, MDN) and applied flexibly.
  3. Proxy Router: To maximize cost-efficiency and collection success rates, the router filter escalates sequentially from No Proxy ➡️ Datacenter Proxy ➡️ Residential Proxy.
  4. Reader Mode & Output Options: Supports modes like article and product depending on the content type. Users can precisely control whether to include images or links in the output.

API Endpoints

AI Reader API

Method Path Description
GET /v1/reader Text conversion and analysis using URL parameters
POST /v1/reader Advanced options (cookies, headers, proxy) via JSON body
POST /v1/reader/chunk Text chunking via an independent TextProcessor microservice

POST /v1/reader Request Example

POST /v1/reader
Content-Type: application/json
X-RapidAPI-Key: <your-rapidapi-key>

{
  "url": "https://example.com",
  "mode": "article",
  "tokenizer": "openai",
  "output": {
    "content": true,
    "metadata": true,
    "tables": true,
    "codeBlocks": true,
    "images": false,
    "links": false
  }
}
