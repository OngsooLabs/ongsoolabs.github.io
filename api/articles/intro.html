Introduction to OngsooLabs API

Welcome to the official documentation for the OngsooLabs API ecosystem.

OngsooLabs API is an enterprise-grade backend platform designed as a comprehensive Knowledge Ingestion Gateway. Our architecture isolates distinct backend services—each highly specialized in data extraction, processing, and delivery—to provide robust, scalable data pipelines for modern AI applications.

Our Vision

In the rapidly evolving landscape of Large Language Models (LLMs) and Retrieval-Augmented Generation (RAG), the quality and structure of data are paramount. Our ultimate vision is to create a centralized AI Knowledge Platform that handles the heavy lifting of data preprocessing.

By providing pristine, AI-ready data pipelines, we empower developers to build smarter, more reliable AI systems without worrying about infrastructure management.

Strategic Roadmap

To realize our vision of a comprehensive Knowledge Ingestion Gateway, we are evolving the platform through the following strategic phases:

  • Phase 1: Foundation (Current)
    • Focus on robust Web-to-Markdown conversion, boilerplate removal, and core metadata extraction via our flagship AI Reader API.
  • Phase 2: Data Utilities
    • Introduction of lightweight tools for text manipulation, HTML cleaning, standalone tokenization, and embedding cost estimation.
  • Phase 3: Automated Monitoring
    • Implementation of watchdog services for sitemaps and RSS feeds to automate change detection and trigger webhook notifications.
  • Phase 4: Advanced Extraction
    • Deep structural extraction pipelines capable of pulling schema-based product information, specific entities, and highly structured JSON from diverse sources.

Platform Architecture

To achieve massive scalability and maintainability, the OngsooLabs API ecosystem is built upon a strict multi-service architecture:

  • Isolated Microservices: Each API service is developed in an isolated environment within a monorepo, strictly adhering to Clean Architecture principles (Domain, Infrastructure, WebApi) with zero-dependency core layers.
  • Independent Scaling & Deployment: Services are fully containerized via Docker and deployed with their own independent CI/CD pipelines, ensuring heavy computational tasks (like headless web rendering) do not bottleneck lightweight utility requests.
  • Shared Platform Services: Common technical utilities are decoupled into shared containerized platform services communicating securely over private internal networks via HTTP/JSON or gRPC.

Available APIs

Currently, we offer the following public-facing service:

  • AI Reader API: Our flagship data ingestion service for converting heterogeneous web content into AI-ready Markdown.

Explore the API documentation from the sidebar to get started, or visit our issue tracker to request new features and integrations as we roll out our strategic roadmap.
