AI Reader API (OngsooLabs.Api v1.0)

A REST API service for AI preprocessing and web content refinement based on .NET 10. It is automatically deployed to AWS EC2 via GitHub Actions CI/CD and served over HTTPS through an Nginx reverse proxy.

Base URL: https://api.ongsoolabs.com

Introduction

The AI Reader API is an AI preprocessing service that transforms web pages into an optimal format ready for immediate use by Large Language Models (LLMs).

Core Value & Pipeline

Web Page → Content Extraction → Boilerplate Removal → Markdown Conversion → Metadata Analysis → Token Estimation → AI Ready Output

Key Features

  • Markdown Conversion: Strips away styling and unnecessary elements from web pages, converting them into pure markdown.
  • Content & Title Extraction: Filters out noise to extract the core text and title of actual articles or products.
  • Language & Metric Analysis: Supports language detection, word count, and token count calculations for the target page.
  • Embedding Cost Estimation: Estimates embedding costs based on the size of the extracted text.
  • Reading Time Calculation: Provides an estimated reading time based on average reading speeds.

System Architecture (v1.0 Improvements)

1. Browser & Context Pool

Instead of launching a heavy browser for every request, a single Chrome instance runs continuously from server startup. It pools isolated contexts and pages to support ultra-fast rendering. A built-in browser recycling mechanism manages memory leaks.

2. Site Rule Engine

Extraction strategies are separated by domain (e.g., Amazon, MDN) and applied flexibly using the ISiteRule interface. When rules for a new domain are needed, they can be expanded simply by adding rules without modifying the main engine.

3. Proxy Router

To maximize cost-efficiency and collection success rates, proxies are not statically assigned. Instead, the router filter escalates sequentially from No Proxy ➡️ Datacenter Proxy ➡️ Residential Proxy depending on the target or block status.

4. Reader Mode & Output Options

  • Reader Mode: Supports article, documentation, product, forum, and generic modes depending on the content type. Content extraction syntax and converters operate independently for each mode.
  • Output Options: Users can precisely control whether to include images, links, tables, and code blocks in the markdown output to reduce response size and save tokens.

API Endpoints

1. Common System API

Method Path Auth Description
GET / ❌ Service identification info
GET /health ❌ ASP.NET health check
GET /ping ❌ Lightweight liveness probe

2. AI Reader API

Method Path Auth Description
GET /v1/reader ✅ Text conversion and analysis using URL parameters
POST /v1/reader ✅ Advanced options (cookies, headers, proxy) via JSON body
POST /v1/reader/chunk ✅ Text chunking via an independent TextProcessor microservice

POST /v1/reader Request Example

POST /v1/reader
Content-Type: application/json
X-RapidAPI-Proxy-Secret: <secret>

{
  "url": "https://example.com",
  "mode": "article",
  "tokenizer": "openai",
  "output": {
    "content": true,
    "metadata": true,
    "tables": true,
    "codeBlocks": true,
    "images": false,
    "links": false
  }
}

Local Development

# 1. Clone the repository
git clone https://github.com/OngsooLabs/OngsooLabs.Api.git

# 2. Run locally
cd src/Services/Scraper/OngsooLabs.Scraper.WebApi
dotnet run

In the Development environment, you can access the Swagger UI at http://localhost:<port>/swagger.
