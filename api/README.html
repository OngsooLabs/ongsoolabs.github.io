AI Reader API (OngsooLabs.Api v1.0)

.NET 10 기반 AI 전처리 및 웹 콘텐츠 정제 REST API 서비스. GitHub Actions CI/CD로 AWS EC2에 자동 배포되며, Nginx 리버스 프록시를 통해 HTTPS로 서빙됩니다.

Base URL: https://api.ongsoolabs.com

서비스 소개

AI Reader API는 웹페이지를 LLM이 바로 사용할 수 있는 최적의 형태로 변환하는 AI 전처리 서비스입니다.

핵심 가치 및 동작 구조

Web Page → Content Extraction → Boilerplate Removal → Markdown Conversion → Metadata Analysis → Token Estimation → AI Ready Output

주요 제공 기능

  • Markdown 변환: 웹페이지의 스타일과 불필요 요소를 걷어내고 순수 마크다운으로 변환합니다.
  • 본문 및 제목 추출: 노이즈를 필터링하고 실제 기사 또는 상품의 핵심 본문 및 제목을 추출합니다.
  • 언어 및 지표 분석: 대상 페이지의 언어 분석, 단어 수 계산, 토큰 수 계산을 지원합니다.
  • Embedding 비용 추정: 추출된 텍스트 크기에 기초한 임베딩 비용을 추정합니다.
  • 독서 시간(Reading Time) 계산: 평균 독서 속도를 기반으로 예상 시간을 제공합니다.

시스템 아키텍처 (v1.0 개선 사항)

1. Browser & Context Pool

매 요청마다 무겁게 브라우저를 띄우지 않고, 초기 서버 구동 시 단일 크롬 브라우저를 상시 구동하여 격리된 컨텍스트와 페이지를 풀링하여 초고속 렌더링을 지원합니다. 주기적인 브라우저 리사이클 메커니즘을 내장하여 메모리 누수를 관리합니다.

2. Site Rule Engine

도메인별(예: 쿠팡, 아마존, MDN 등)로 수집 전략을 분리하여 ISiteRule 인터페이스 기반으로 유연하게 규칙을 적용합니다. 신규 도메인의 정책이 추가되더라도 메인 엔진의 수정 없이 룰 추가만으로 확장이 가능합니다.

3. Proxy Router

비용 효율과 수집 성공률의 극대화를 위해 프록시를 항상 고정하여 사용하지 않고, 수집 대상이나 차단 상황에 맞추어 No Proxy ➡️ Datacenter Proxy ➡️ Residential Proxy 순으로 순차 에스컬레이션하는 로터 필터를 지원합니다.

4. Reader Mode & Output Options

  • Reader Mode: 콘텐츠의 형태에 따라 article, documentation, product, forum, generic 모드를 지원하며, 본문 추출 구문 및 변환기가 독립적으로 동작합니다.
  • Output Options: 응답 데이터 크기를 줄이고 토큰을 절약할 수 있도록 이미지, 링크, 테이블, 코드 블록 등의 마크다운 포함 여부를 사용자가 정밀하게 제어할 수 있습니다.

API 엔드포인트

1. 공통 시스템 API

메서드 경로 인증 설명
GET / ❌ 서비스 식별 정보
GET /health ❌ ASP.NET 헬스체크
GET /ping ❌ 경량 생존 확인

2. AI Reader API

메서드 경로 인증 설명
GET /v1/reader ✅ URL 파라미터를 사용한 텍스트 변환 및 분석
POST /v1/reader ✅ JSON 바디를 통한 고급 옵션(쿠키, 헤더, 프록시) 제공
POST /v1/reader/chunk ✅ 독립된 TextProcessor 마이크로서비스를 경유한 텍스트 청킹

POST /v1/reader 요청 예시

POST /v1/reader
Content-Type: application/json
X-RapidAPI-Proxy-Secret: <secret>

{
  "url": "https://example.com",
  "mode": "article",
  "tokenizer": "openai",
  "output": {
    "content": true,
    "metadata": true,
    "tables": true,
    "codeBlocks": true,
    "images": false,
    "links": false
  }
}

운영 정책 및 이용약관 (Terms & Policies)

AI Reader API를 이용하는 모든 사용자는 다음 운영 정책을 반드시 준수하고 이에 동의해야 합니다.

  1. 이용자 책임의 원칙: 사용자는 AI 전처리 기능을 통해 변환하고자 하는 대상 웹사이트의 서비스 이용약관, 저작권 규정 및 관련 법령을 전적으로 준수해야 합니다.
  2. 서비스 안정성 통제: 운영자는 서비스 전체의 안정성과 악의적인 자원 고갈(Abuse) 방지를 위해 특정 계정의 요청 속도를 제한하거나 접근을 차단할 수 있습니다.
  3. 결과 보장의 한계: 인터넷상의 모든 웹사이트에 대한 완벽한 접근 및 추출 정확도를 보장하지 않으며, 대상 사이트의 구조 변경이나 보안 정책에 따라 결과는 유동적일 수 있습니다.
  4. 결과물 활용 및 소유권: 서비스는 웹 데이터의 전처리 및 포맷 변환 기능만을 기계적으로 대행하며, 가공 및 파이프라인(LLM, RAG 등) 적용 결과물의 상업적 활용에 대한 모든 법적 책임은 이용자에게 있습니다.

로컬 개발 및 실행

# 1. 리포지토리 클론
git clone https://github.com/OngsooLabs/OngsooLabs.Api.git

# 2. 로컬 실행
cd src/Services/Scraper/OngsooLabs.Scraper.WebApi
dotnet run

Development 환경에서 http://localhost:<port>/swagger로 Swagger UI에 접근할 수 있습니다.
