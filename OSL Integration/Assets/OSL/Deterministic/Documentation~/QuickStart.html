Quick Start

This document walks through the most common usage patterns of OSL Deterministic Core. Every snippet is intentionally minimal and self-contained.

1. Create an RNG

using OSL.Deterministic.Runtime.Core;

// Seed-only: identical seed produces identical sequences.
var rng = new OSLRandom(seed: 0xC0FFEE);

// Seed + stream id: useful to derive independent sub-RNGs from one master seed.
var combatRng = new OSLRandom(new PCG32(seed: 0xC0FFEE, stream: 1));
var lootRng   = new OSLRandom(new PCG32(seed: 0xC0FFEE, stream: 2));

Never call UnityEngine.Random or System.Random inside deterministic code. Their results vary between platforms and Unity versions.

2. Basic Draws

uint  u  = rng.NextUInt();           // [0, 2^32)
uint  u2 = rng.NextUInt(100u);       // [0, 100)   unbiased
int   i  = rng.NextInt(10, 20);      // [10, 20)
float f  = rng.NextFloat();          // [0, 1)
bool  b  = rng.NextBool();
bool  c  = rng.Chance(0.15f);        // 15% true
int   r  = rng.Range(1, 7);          // d6

3. Save & Restore State (zero alloc)

Span<byte> buf = stackalloc byte[rng.StateSize];   // 16 bytes for PCG32
rng.SaveState(buf);

// Later — same process, same machine, or another machine entirely.
rng.LoadState(buf);

Use this to:

  • Implement rollback netcode (snapshot RNG along with simulation state).
  • Persist mid-run state across save files.
  • Reproduce a bug from a player's session by serializing the engine state.

4. Fisher–Yates Shuffle

using OSL.Deterministic.Runtime.Shuffling;

int[] deck = new int[52];
for (int i = 0; i < deck.Length; i++) deck[i] = i;

OSLDeckShuffler.Shuffle(deck, rng);            // in-place, zero alloc
OSLDeckShuffler.Shuffle(deck, rng, 0, 10);     // shuffle only the first 10 elements

5. Weighted Gacha

using OSL.Deterministic.Runtime.Gacha;

var table = new OSLGachaTable<GachaEntry<string>>(new []
{
	new GachaEntry<string>(60, "common",    "C"),
	new GachaEntry<string>(30, "uncommon",  "U"),
	new GachaEntry<string>(9,  "rare",      "R"),
	new GachaEntry<string>(1,  "legendary", "L"),
});

GachaEntry<string> single = table.Roll(rng);

var buffer = new GachaEntry<string>[10];
table.RollMany(rng, buffer);                   // 10-pull

With Pity

using OSL.Deterministic.Runtime.Pity;

var pity = new SimpleHardPity(threshold: 50, rareIndex: 3); // legendary at index 3

table.RollMany(rng, pity, buffer);             // pity forces index 3 after 50 misses

From JSON (designer-authored tables)

string json = System.IO.File.ReadAllText("Assets/Tables/legendary_banner.json");
var def     = UnityEngine.JsonUtility.FromJson<GachaTableJson>(json);
var loaded  = def.ToTable();

From ScriptableObject (inspector-authored tables)

Create > OSL > Deterministic > Gacha Table produces an asset whose ToTable() method returns a ready-to-use OSLGachaTable.

6. Combat Probabilities

using OSL.Deterministic.Runtime.Combat;

var combat = new OSLCombatResolver(rng);

bool crit    = combat.RollPermille(150);       // 15.0%
bool dodge   = combat.RollPercent(20f);
bool special = combat.RollProbability(1, 8);   // exactly 1-in-8

int damage   = combat.RollDice(sides: 6, count: 2);   // 2d6
bool success = combat.RollCritical(criticalChancePermille: 80,
								   failureChancePermille: 50);

Prefer the integer-typed APIs (RollPermille, RollProbability) when you need to guarantee identical results across platforms — floating-point comparisons can drift on different CPUs.

7. Deriving Sub-RNGs Deterministically

ulong masterSeed = 0xC0FFEE;
var combat = new OSLRandom(new PCG32(masterSeed, stream: 1));
var loot   = new OSLRandom(new PCG32(masterSeed, stream: 2));
var ai     = new OSLRandom(new PCG32(masterSeed, stream: 3));

Different stream values produce statistically independent sequences from the same seed — ideal for isolating subsystems so adding a new combat roll does not desync loot or AI.

8. Patterns to Avoid

  • System.Random, UnityEngine.Random, Guid.NewGuid() inside deterministic code.
  • DateTime.Now, Stopwatch.GetTimestamp() as seeds for replay-bound systems.
  • Dictionary<TKey, TValue> enumeration order for game logic (order is not guaranteed across runtimes).
  • float/double accumulation across many frames as a source of decisions — prefer integer counters or fixed-point math.

9. Verifying Determinism

The bundled Lockstep Demo sample runs two independent simulations side by side and shows a per-tick state hash. If you fork the runtime or add custom modules, extend that demo to cover your code path — divergence will be visible immediately.
