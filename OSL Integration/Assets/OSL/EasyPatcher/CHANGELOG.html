Changelog

All notable changes to this package are documented here. The format is based on Keep a Changelog.

[1.0.1]

Fixed

  • OSLPatchManager: HMAC verification now runs whenever the manifest declares an hmacHash, regardless of the encrypted flag. Missing keys are reported as a hard failure instead of silently passing.
  • OSLPatchManager: unified local cache path with the loader (persistentDataPath/OSL/EasyPatcher); per-file download timeout removed for large files; progress is clamped to [0,1].
  • OSLPatchUIController: Start() is wrapped in try/catch so a failed patch flow no longer leaves the UI in an undefined state.
  • OSLUploaderWindow / OSLSimpleBundleLoader: AES is pinned to CBC + PKCS7 on both encrypt and decrypt paths.
  • PatchTestRunner: hash-mismatch scenario now skips only local_version.json, so corruption is applied to a real payload file.
  • PatchManifest: files list is initialized; PatchFileInfo carries fileName, sizeBytes, sha256Hash, encrypted, and hmacHash consistently across editor and runtime.
  • OSL.EasyPatcher.Runtime.asmdef: corrected TMP versionDefines wiring.

Added

  • package.json for UPM compatibility.
  • README Publishing section describing .unitypackage export and UPM install flows.

[1.0.0] - 2025-01-01

Added

  • Initial release.
  • Strategy-pattern runtime loader (IOSLLoader) and OSLSimpleBundleLoader.
  • Streaming, hash-verified OSLPatchManager with optional HMAC verification.
  • Editor uploader window with Upload / Download / Versions tabs.
  • Hierarchical tree view with per-file/folder encryption and tamper-proof toggles.
  • Versioned patch slots.
  • Upload backends: Local Folder, Google Cloud Storage, AWS S3, Unity CCD.
  • TMP-friendly OSLPatchUIController with automatic legacy UI.Text fallback.
  • Sample PatchTestRunner covering 5 end-to-end scenarios.
  • Assembly definitions for clean Runtime / Editor / Sample separation.
